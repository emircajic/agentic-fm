Framework is built and lives in agent/library/Framework/. Apply it to all new scripts.

**Why:** Standardize script architecture across solutions for consistency, debuggability, and reuse.

**How to apply:** Every new script uses Params.parse as first line, Response_* CFs for envelope, Log.add for logging. Root orchestrators flush $$~LOG via UTIL__FlushLog on exit.

## Key decisions

- **Envelope:** {status, data, messages:{error[], warning[], info[]}}
- **Status:** auto-derived by Response_Finalize from message counts (never set manually)
- **data on fail:** wiped by Response_Finalize unless $$~DEBUG = 1
- **Params.parse:** auto-detects JSON/pipe/key=value, unpacks $variables, sets $traceID and $isRoot
- **Trace ID:** generated by root orchestrator (absent in params = root), forwarded to all sub-scripts via traceID param key
- **Logging:** Log.add CF accumulates to $$~LOG (zero I/O), root orchestrator flushes to UTIL__ScriptLog table via Data API on exit (always, success or fail)
- **Flush:** $isRoot = True triggers flush — detected automatically by Params.parse
- **Error propagation:** fail-fast = Exit Script [ $r ]; multi-step collection = Response_Absorb($r)

## Reader CFs built (Framework/Functions/Response/)
Response_IsOk, HasErrors, HasWarnings, GetStatus, GetData, GetErrors, GetWarnings, GetInfos, GetErrorCount, GetWarningCount, FirstError, FirstErrorCode, Absorb
